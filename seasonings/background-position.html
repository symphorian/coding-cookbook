Basic Code Snippets

background-position: right;
background-size: bottom left;
background-size: center center;

Additional Examples

The background-position property sets the position of the container's background image. The values provided to the property will "anchor" the image to a particular side, corner, or position within its container.

Below is an image we'll be using for a few examples.

An <img> element

The background-position property is always used with the background-image property. After all, if there is no image in the background, there is nothing to position! We're going to assume the background-image property is set in the following examples, and take a look at some examples for how background-position can be used.

The Position Keywords ('top', 'bottom', 'left', 'right', and 'center')

The easiest way to use the background-position property is to use the position keywords. Four of these correspond to the different sides of the Box Model (left, right, top, and bottom) and the last one is 'center'.

These are fairly intuitive to use...when the image is smaller than the container. Things get more complicated when the image is larger than its container...but we'll look at that later.

However, please keep in mind...THIS PROPERTY IS FOR BACKGROUND IMAGES ONLY! You cannot use it with regular old <img> elements. What you see in the examples below is a BACKGROUND IMAGE being moved around, NOT an <img> element!

Example 1: background-position: left;
background-image: url("../images/axolotl.webp");
background-position: left;
height: 400px;
Example 2: background-position: right;
background-image: url("../images/axolotl.webp");
background-position: right;
height: 400px;
Example 3: background-position: top;
background-image: url("../images/axolotl.webp");
background-position: top;
height: 400px;
Example 4: background-position: bottom;
background-image: url("../images/axolotl.webp");
background-position: bottom;
height: 400px;

But not only can you use the sides individually, you can also combine them together. And when you throw the 'center' keyword into the mix, you have a total of nine different locations you can anchor your image to.

Here are a few more examples:

Example 5: background-position: top right;
background-image: url("../images/axolotl.webp");
background-position: top right;
height: 400px;
Example 6: background-position: bottom left;
background-image: url("../images/axolotl.webp");
background-position: bottom left;
height: 400px;
Example 7: background-position: center center;
background-image: url("../images/axolotl.webp");
background-position: center center;
height: 400px;
Positioning Background Images Sized with 'cover'

The background-position property is fairly intuitive with small images...but what about when the image covers the entire container? What about when some parts of it are cropped off?

Then things get a bit more confusing visually, but the background images actually still behave the same way. Let's look at the default behavior of background-position first when background-size: cover; is applied.

Example 8: background-position and background-size
background-image: url("../images/axolotl.webp");
background-size: cover;
height: 400px;

In the example above, the image fills the container horizontally, but it can't fit vertically, so something needs to be cropped off. By default, it is anchored to the top left corner, as if background-position: top left; was applied, so the bottom of the image gets chopped off.

What if we wanted it to be anchored somewhere else? Like the bottom, or the center? Well, take a look at the next few examples, and you'll see we can do exactly that.

Example 9: background-size: cover; and background-position: bottom;
background-image: url("../images/axolotl.webp");
background-size: cover;
background-position: bottom;
height: 400px;
Example 10: background-size: cover; and background-position: center;
background-image: url("../images/axolotl.webp");
background-size: cover;
background-position: center;
height: 400px;

The same would apply if you used the 'left' and 'right' values as well, but only if the container was taller and thinner than the image you were using as a background.

Sometimes, our container could be different aspect ratios depending on the width of the screen. In that case, you may need to be clever about your background image choices, or use Media Queries to reset the background-position property based on the width of the screen.
