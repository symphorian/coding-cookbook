Basic Code Snippets

background: linear-gradient(red, blue);
background: linear-gradient(to bottom right, yellow, orange, purple);
background: linear-gradient(45deg, #00000000, #000000FF);
background: linear-gradient(red, white 30%, blue 50%);

Additional Examples

Let's be very clear to start: linear-gradient is NOT a CSS Property!

linear-gradient is a function that can be passed as a value to the background property (well, technically background-image, but we can cheat and use the shorthand property).

If you try to use linear-gradient as its own property, you will get nothing but broken code.

Now, with that out of the way, let's look at some examples.

Two color stops

Gradients are created with values known as "color stops"; color stops essentially tell the gradient what colors to transition between.

One of the simplest color gradients is one between two colors, so let's do that below with the following code:

background: linear-gradient(yellow, green);

Two color stops with direction

Above, we created a linear gradient that goes from yellow to green, from the top of the element's background to the bottom. What if we wanted that gradient to go in a different direction?

In that case, we can specify the angle of the gradient by the side or corner that we want the gradient to start from by using the special side keywords of "top", "bottom", "left", and "right", as in the examples below.

background: linear-gradient(to top, yellow, green);

background: linear-gradient(to right, yellow, green);

background: linear-gradient(to bottom left, yellow, green);

We can also directly specify the angle we want using degrees instead.

background: linear-gradient(45deg, yellow, green);

background: linear-gradient(-90deg, yellow, green);

Multiple color stops

Theoretically, there's no limit to the number of color stops you could give to a gradient (though practically, you'd never need THAT many color stops).

Here's an example with multiple color stops:

background: linear-gradient(90deg, blue, purple, lightblue, aquamarine, teal);

Color Stops with Length

We already have a lot of flexibility with linear gradient, but what if we wanted even more control?

For instance, what if we wanted to create a gradient that stopped transitioning in the middle of the element?

In that case, we could provide a length as a percentage value after a color stop to tell it, well, where that color should "stop" on the gradient. Let's see it in action.

background: linear-gradient(90deg, crimson, gold 50%);

See how the gold color transition stops in the middle of the gradient at 50%? That's because of the color stop percentage that we added.

You do have to be a little thoughtful with color stop lengths, or else they may not work properly. As you progress through the gradient, your color stop percentages should always be increasing, since they're referring to points on the gradient.

For instance, it wouldn't make sense to provide color stop lengths in this order: 20%, 80%, 50%.

By the time you've reached the color stop at 80%, you've already passed 50%, and you can't go backwards in a gradient.

Here's another example of color stops with lengths:

background: linear-gradient(90deg, red 20%, white, blue 60%);

In this case, red only starts transitioning to white at 20%, and white must transition to blue by 60%, so that limits the transition from red to white to blue between 20% and 60% of the gradient.

Kinda ends up looking like the Pepsi color scheme, doesn't it?
