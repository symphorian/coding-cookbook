Basic Code Snippets

grid-area: area-1;
grid-area: header;
grid-area: name-of-area;

Additional Examples

NOTE: The grid-area property is a Grid Item Property.

This property will have no effect if applied to an element that is not INSIDE an activated Grid Container (i.e. has display: grid; applied to it)

The grid-area property assigns a grid area to a Grid Item inside of a Grid Container.

The grid-area property will only have an effect if used on Grid Items inside of a Grid Container with grid-template-areas set on it.

And even then, if the area name given to grid-area doesn't match up EXACTLY to a corresponding area in its Grid Container's grid-template-areas property, grid-area will have no effect.

If you do not know how the grid-template-areas property works, it is strongly recommended you read up on it next if you intend to use grid-area!

An Analogy: Rooms of a House

As the grid-area property works in tandem with the grid-template-areas property, we will the same analogy we used to describe that property as well.

It may help to think of this property as enabling you to map out the rooms of a house.

When the house is empty, you can think "alright, this room is going to be the living room, this room is going to be a bedroom, this room is going to be a dining room..." and so on.

Then, based on those room designations, you start filling the rooms with furniture that fits the room type you decided on.

When you decide that "this room is going to be the bedroom", that is more or less the equivalent of setting the grid-area property on a Grid Item inside of a Grid Container.

Then whatever is inside of that Grid Item becomes the "furniture" in that area of the Grid Container's grid-template-areas property.

Example

We'll use the same example that we used in the grid-template-areas property entry in the Coding Cookbook, just focusing more on the grid-area side.

Suppose we want a grid layout with empty gutters on the sides, along with a header, a sidebar, and two content areas.

That might look something like this, using grid-template-areas:

grid-template-areas:
    ". header header ."
    ". sidebar content-1 ."
    ". sidebar content-2 .";

The code to assign these areas to the Grid Items inside of the Grid Container with that grid-template-areas code would look something like this (with highlights matching areas above with those below):

.header-sidebar-layout .box-a {
    grid-area: header;
}

.header-sidebar-layout .box-b {
    grid-area: sidebar;
}

.header-sidebar-layout .box-c {
    grid-area: content-1;
}

.header-sidebar-layout .box-d {
    grid-area: content-2;
}

Remember that the area names given to grid-area must match EXACTLY with the names used in grid-template-area; even one small case difference like Area vs. area will break the connection between them.

As you can see, we've completely reorganized the Grid Items using grid-template-areas! Let's look at those areas we defined again to see how they match up with the result above (we added a little extra formatting to the code so they line up more nicely):

grid-template-areas:
    ". header header ."
    ". sidebar content-1 ."
    ". sidebar content-2 .";

See how our grid-template-areas property is basically the textual, visual representation of how our Grid Items look once they're plugged in?

A is the header, B is the sidebar, C is content-1, D is content-2, and the dots represent the empty gutters on the left and right sides of the Grid Items.

And that is due to the code we looked at above, assigning those areas into the Grid Items (again, with matching highlights):

.header-sidebar-layout .box-a {
    grid-area: header;
}

.header-sidebar-layout .box-b {
    grid-area: sidebar;
}

.header-sidebar-layout .box-c {
    grid-area: content-1;
}

.header-sidebar-layout .box-d {
    grid-area: content-2;
}

Now just imagine how powerful this becomes when we put actual content inside the grid, and not just empty boxes!
