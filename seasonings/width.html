Basic Code Snippets

width: 500px;
width: 100%;
width: 50%;

Additional Examples

The width property, as you might be able to guess, allows you to specify a width for a particular element.

However, there are actually a couple different types of units that can be used with the width property, along with a related property that you should know about.

So let's take a look at how we can use the width property, as well as some of the caveats associated with it.

Width and Document Flow

The width property might behave differently than you expect depending on the type of element you use it with. Let's take a look at some examples.

Width and Block Elements

The width property is pretty straightforward with block elements. Let's apply set it with some pixel values to a few div elements along with a border and background.

width: 150px;
width: 400px;
width: 250px;
width: 30px;

Pretty much as expected, each block element is set to the pixel widths we set for them.

But notice a couple of things:

  • Block elements always stay on their line; they never go side-by-side, even when there is room to do so.
  • When block elements' width gets too short, the content in them may overflow the width we specified.
  • When block elements' width gets longer than their container, the block element will overflow past the edge of the container.

Now let's take a look at some examples using relative units, such as percentages. It's important to remember that when using relative units, they are usually relative to their containers.

width: 100%;
width: 50%;

This also looks pretty straightforward at first glance: one block fills 100% of the width of the container, while the other fills 50% of it.

...but if that's the case, why are there gaps on the sides of the 100% block?

Part of the answer is because the container has padding set on it, so there should be space INSIDE the container before the content begins.

But if that was it, then why are the gaps not equal on each side?

Well, that has to do with a rather annoying default behavior of HTML Elements...

Box-sizing and content-box vs border-box

There is a CSS Property called box-sizing which has a default value of 'content-box'.

And 'content-box' means that when we set a width property, we are setting the width of the CONTENT area of the element.

That means that the padding and border widths get added on IN ADDITION to the width that we specified. Which why there's less space on the right side.

Almost no one thinks that way or expects that when we set a width on an element, so we usually change the box-sizing property to a value of 'border-box'.

Let's try that with our previous example, and see what happens.

width: 100%; (with box-sizing: border-box;)

Now the space on either side is equal, and that's because the width we set includes the border and padding on our box, thanks to the 'border-box' value.

So, moral of the story: set the box-sizing property to 'border-box' if you plan on using relative width values like percentages.

You'll save yourself a lot of headaches.

Width and Inline Elements

The width property...is basically ignored by inline elements. Check out the examples below for proof.

width: 150px; width: 400px; width: 250px;

Since inline element ignore width, we don't need to linger. Let's see how inline-block elements compare...

Width and Inline-Block Elements

Unlike inline elements, inline-block elements behave nicely with the width property.

width: 150px;
width: 400px;
width: 250px;
width: 30px;

They actually behave very similar to the block elements, but unlike the block elements, if multiple inline-blocks can fit on the same line, they'll slide up next to each other, side-by-side.

Aside from that, inline-blocks are nearly identical to block elements in terms of how they respect the width property.

width: 100%;
width: 50%;

...which includes the same box-sizing issue. Thankfully, we can resolve it the same way, just by adding box-sizing: border-box; to our elements with width set.

width: 100%;
