Basic Code Snippets

background-size: cover;
background-size: contain;
background-size: 30%;

Additional Examples

The background-size property sets the size of the container's background image. The image can be left to its natural size, stretched, or constrained to fit the available space.

Below is an image we'll be using for a few examples.

An <img> element

The background-size property is always used with the background-image property. After all, if there is no image in the background, there is nothing to size! We're going to assume the background-image property is set in the following examples, and take a look at some common values used with background-size.

The 'cover' keyword

The most common value used with background-size (at least in this course) is the 'cover' value. This special keyword tell the background image to literally "cover" the entirety of the container's background, even if that means cropping out parts of the image if its aspect ratio doesn't match that of the container.

Example 1: The background-size property used with 'cover'
background-image: url("../images/axolotl.webp");
background-size: cover;
height: 400px;

However, as you can see, sometimes that is less than ideal. Our cute little axolotl in the example above has almost been cropped out of the container!

To fix this, we often use a related property called background-position to adjust what part of the image should be anchored in view. Most commonly, we just center the image horizontally and vertically, as in the example below.

Example 2: The background-size property used with 'cover' and the background-position property.
background-image: url("../images/axolotl.webp");
background-size: cover;
background-position: center center;
height: 400px;

In this case, you can see that the image is still cropped, but it is the edges of the images that are cropped, not just the bottom of the image.

The 'contain' keyword

The 'contain' value is essentially the opposite of 'cover'. Rather than making sure that the container is "covered" by the image, the 'contain' keyword says to make sure that the image is displayed in its entirety with no cropping at least once within the container, even if that means parts of container are left empty or the image starts repeating.

Example 3: The background-size property used with 'contain'
background-image: url("../images/axolotl.webp");
background-size: contain;
height: 400px;

You can see that by default, when 'contain' is used, the image starts repeating or "tiling" to fill the rest of the container.

If we don't want that behavior, we can add in another property called background-repeat and set it to 'no-repeat' to turn off the tiling behavior.

Example 4: The background-size property used with 'contain', with background-repeat and background-color
background-image: url("../images/axolotl.webp");
background-size: contain;
background-repeat: no-repeat;
background-color: grey;
height: 400px;

With the repeating (or "tiling") behavior turned off, the rest of the container not painted by the image is left empty. The blank space can be filled by the background-color property if one is provided.

Using width values

Though less common, you can also provide background-size with regular old width values, using either pixels or percentages.

Width values are useful for when you have a background image that you WANT repeated, and want it to render at a specific size.

For example, suppose we use the image below as a background-image:

An <img> element of a pawprint pattern

The image above is designed to be repeated in a background, and if we use a width value in background-size, we can scale the image however we want, and it will repeat seamlessly across the background, so that you can't even tell where the actual image begins and ends. (...however, make sure that you provide good contrast for any text placed on top of a background like this one!)

Example 5: The background-size property used with a width value
background-image: url("../images/axolotl.webp");
background-size: 250px;
height: 400px;
