Basic Code Snippets

grid-template-areas:
    "area-1 area-2 area-3";

grid-template-areas:
    "header header header"
    ". content ."
    "footer footer footer";

Additional Examples

NOTE: The grid-template-area property is a Grid Container Property.

This property will have no effect if applied to an element that is not an activated Grid Container (i.e. has display: grid; applied to it)

The grid-template-area property defines areas in a grid template, based on the columns defined by grid-template-columns.

These areas must then be assigned to Grid Items, and those Grid Items will be slotted into those areas in the Grid Container.

An Analogy: Rooms of a House

It may help to think of this property as allowing you to map out the rooms of a house.

When the house is empty, you can think "alright, this room is going to be the living room, this room is going to be a bedroom, this room is going to be a dining room..." and so on.

Then, based on those room designations, you start filling the rooms with furniture that fits the room type you decided on.

The grid-template-areas property is similar, but instead of rooms, you have grid cells. And instead of furniture, you have HTML elements (technically Grid Items).

Syntax

The grid-template-areas property has a very strange syntax, probably the weirdest you'll see in this entire course.

You'll specify grid areas by rows, and the rows of areas MUST be contained in double quotes.

The areas themselves must be separated by spaces within the row (the double quotes).

This is easiest to see in practice, so we'll dive right into our first exampel

Simple Example: Three Areas

We'll start with something basic: a grid with only three columns set, and three Grid Items inside of the Grid Container.

By default, Items A, B, and C are slotted into the columns one at a time; each Item takes up one cell (or room, to use the house analogy) in the Grid.

This is kind of like a moving company moving furniture into the house while you're not there: The movers have no idea what you're plans are, they just dump the furniture where there's room as they unload it.

Now, let's be a bit more explicit about where we want our Grid Items (i.e. furniture) to go.

grid-template-areas:
    "area-c area-a area-b";

However, just adding the code above won't change anything, as you can see.

There's a second step to using grid-template-areas: you also need to assign the grid areas to the Grid Items.

This can be done with code similar to the snippet below, repeated for each Item in the Grid.

(NOTE: grid-area is a Grid Item property; it will have no effect if used on elements without a parent that has display: grid; applied to it.)

grid-area: area-c;

In that code snippet, we're assigning a Grid Item the grid-area of area-c from grid-template-areas. We probably want to assign that to Grid Item C.

If you want a closer look at how the grid-area property works, check out that page in the Coding Cookbook next!

That is the power of grid-template-areas: we didn't change our HTML at all, but with CSS, we can completely reorganize and reorder the Grid Items any way we want!

Complex Example: Dots and Spanning Multiple Cells

Now we'll introduce a more complex (and more practical) example.

Suppose we want a grid layout with empty gutters on the sides, along with a header, a sidebar, and two content areas.

That might look something like this, using grid-template-areas:

grid-template-areas:
    ". header header ."
    ". sidebar content-1 ."
    ". sidebar content-2 .";

Let's analyze what we have in there:

First off, we have three sets of double quotes, which means we have three rows we're defining in the template. (Note that only the last has the semi-colon at the end!)

Inside the double quotes of each of those rows, we have four areas values defined (some are dots (.) and some are repeated; we'll get to those later). That means that we expect the grid template to have four columns in each row (which should have been defined in grid-template-columns).

Now, as for the area values themselves: the first thing that stands out are those dots. A dot is a special value in grid-template-areas: it means that the cell it occupies is meant to be unassigned or empty. This is exactly what we need for our gutters, which should be empty spaces on either side of the main page content.

As for the repeated area values (header and sidebar), this introduces a new concept in grid-template-areas: the idea that we can assign an area ACROSS cells.

In the case of header, it is repeated twice in two adjacent cells of the same row. That means that the header area will take up the space of those two center columns in the top row.

In the case of sidebar, it is repeated twice in two adjacent cells of the same column. That means that the sidebar area will take up the space of those two cells in the second column, across two rows.

As you can see, we've completely reorganized the Grid Items using grid-template-areas! Let's look at those areas we defined again to see how they match up with the result above (we added a little extra formatting to the code so they line up more nicely):

grid-template-areas:
    ". header  header    ."
    ". sidebar content-1 ."
    ". sidebar content-2 .";

See how our grid-template-areas property is basically the textual, visual representation of how our Grid Items look once they're plugged in?

A is the header, B is the sidebar, C is content-1, D is content-2, and the dots represent the empty gutters on the left and right sides of the Grid Items.

Now just imagine how powerful this becomes when we put actual content inside the grid, and not just empty boxes!
