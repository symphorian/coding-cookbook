NOTE: The flex-direction property is a Flexbox Container Property.

This property will have no effect if applied to an element that is not an activated Flexbox Container (i.e. has display: flex; applied to it)

All flexboxes have a flex-direction set; by default, their flex-direction is a row, which is why when you activate flexbox on a container, its contents immediately rearrange into a row, even if you wrote no other flexbox code.

The two most common values provided to flex-direction are row and column.

Example of flex-direction: row;

Again, row is the default flex-direction property value. If you don't set flex-direction yourself, it will automatically be set to row.

The row value will arrange items horizontally, similar to how inline items behave by default.

Main-axis and Cross-axis

Using the flex-direction property is fairly straightforward, but you must be aware that doing so has certain side-effects.

Each Flexbox Container has the concept of a main-axis and a cross-axis, and it is the flex-direction property that determines what these axes are.

The main-axis specifies the direction that content in the flexbox flows.

When you set flex-direction to row, you set the main-axis to be the x-axis.

When you set flex-direction to column, you set the main-axis to be the y-axis.

The cross-axis, on the other hand, is simply the opposite of the main-axis.

So when you set flex-direction to row, you set the cross-axis to be the y-axis.

And when you set flex-direction to column, you set the cross-axis to be the x-axis.

Why is this important? Because certain flexbox properties (justify-content and align-items in particular) change their behavior depending on which axis the main-axis and cross-axis are set to.

So, try to remember the following:

  • flex-direction: row;
    • main-axis: x (horizontal)
    • cross-axis: y (vertical)
  • flex-direction: column;
    • main-axis: y (vertical)
    • cross-axis: x (horizontal)
