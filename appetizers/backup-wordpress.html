Backing Up and Restoring (WordPress)

Save Frequently and Often, so You Don't Lose Your Work!

Backing Up Your WordPress Site

Creating the save file for your site

  1. Saving and backing up your work is important no matter what job field you find yourself in.

    GitHub itself is our backup if we're coding, but when we're in WordPress, we have to save our backups more manually.

    First off, if you haven't already, log in to your WordPress site, and navigate to the Dashboard.

    If you're on the live version of your site, but the Admin Bar doesn't show up at the top of the screen, you might have been logged out due to inactivity.

    Add /login to the end of the URL, and you'll be able to login again.

    Once on the Dashboard, navigate to Tools -> Export.

  2. The Dashboard will switch to the Export view.

    From here, select the content that you want to backup (probably "All content").

    Then click the Download Export File button.

    This will create something called an XML file. This is essentially your backup or save file. Store it somewhere safe!

Congrats! You know how to backup (aka Export) your WordPress site!

Restoring Your WordPress Site

Recovering from Catastrophe

  1. If you're in this section, most likely something went catastrophically wrong with your WordPress site, and for whatever reason, you need to start over from scratch.

    But as long as you exported a save file from the previous section, you should be (mostly) fine!

    First off, if you haven't already, log in to your WordPress site, and navigate to the Dashboard.

    If you're on the live version of your site, but the Admin Bar doesn't show up at the top of the screen, you might have been logged out due to inactivity.

    Add /login to the end of the URL, and you'll be able to login again.

    Once on the Dashboard, navigate to Tools -> Import.

  2. The Dashboard will switch to the Import view.

    There are actually many different ways that you could potentially import content to a WordPress site, but in our case, we're going to use the bottom option.

    In the WordPress section, click the Install Now link.

    It should only take a moment or two to install.

  3. When the WordPress Importer is successfully installed, you'll get a notification banner at the top of the screen.

    Next, click on the Run importer link to move on to the next step of the process.

  4. On the next screen, click the button that says "Choose File".

    A dialog will open up, prompting you to select the XML file on your computer that you want to import to restore the site.

    Select whichever save file you want to use for the import process.

  5. After selecting the file, it will show (part of) the file name on the Import screen.

    Next, click the blue "Upload file and import" button.

  6. On the next screen, you'll be asked how you want to decide the author for the content you're importing.

    Remember, a CMS like WordPress often has multiple people working on the same blog, so choosing which one of them should be the author when importing content is important.

    However, in your case, you're most likely the only author, so you don't need to worry about this too much.

    If you used the same username on this new site as your previous one (the one you're importing content from), you can probably leave it set to the default.

    However, if you did choose a different username, then pick your current username from the dropdown.

    After that, also make sure you check the checkbox under Import Attachments.

    Lastly, you can click the Submit button to finally trigger the Import process.

  7. On the next screen, you'll see the results of the import process.

    The Post and Page content should have copied over with no problems; however, you may need to reupload the images, depending on how messed up your site is/was.

    Hopefully all went well, and you can click on the "Have fun!" link to return to the Dashboard.

Congrats! You know how to restore (aka Import) your WordPress site!
