Utilizing Google Fonts (for Code)

A tutorial showing how to include pre-baked fonts in your web page

Ensuring fonts render the same for every viewer is a surprisingly tricky business. Some operating systems have fonts that others don't, and you have to take that into account when sharing your designs, even in an offline format like a PDF.

There is a feature of CSS called "web fonts" that allow developers to upload their own custom fonts to servers and create new font families for use on their web sites.

However, this process can get a little complex. You still need to worry about licensing, and depending on the font file type, it still may not load correctly in certain browsers.

Thankfully, there are services out there that handle most of this work for us, like Google Fonts. This tutorial is going to show how you can take advantage of Google Fonts to pull fonts from their library into your web page.

Google Fonts

Home to over 1500 web fonts and counting

  1. First off, let's navigate to the Google Fonts home page by clicking on the link below:

    fonts.google.com

    Once you arrive, you'll find a page similar to the screenshot on the right (I have it set to Dark Mode; it may show as Light Mode for you).

    As you can see, there's a lot more than just English fonts available to use!

  2. While you can browse all 1550 fonts available as they choose to display them, it's more likely you have a specific type of font in mind that you want to find.

    In that case, you'll probably want to take advantage of filters like "Decorative Stroke" or "Classification".

    For instance, clicking on "Handwriting" under "Classification" will apply a filter to the search results only showing fonts tagged as handwriting fonts.

    You can select whichever filters you want to search for, and once you do, the search results will update themselves accordingly.

  3. After scrolling through the search results and finding a font that interests you, go ahead and click on the card containing that font.

    In this case, I'm selecting the "Ms Madi" font, as that has the nice, elegant cursive style that I'm looking for.

  4. After selecting a font, you'll be brought to that font's specific page in Google Fonts.

    From here, you'll be able to see a preview of the font, the different Styles available for that font, as well as some previews for how specific styles render at different sizes.

  5. Click the "Get Font" button to add the font to your "shopping bag"

    The screen this button takes you to is the inside of your "shopping bag", where all the other fonts you select get added.

  6. Now we're inside the "shopping bag", where any fonts you've selected will appear. At the moment, we only have one, but we can actually add as many as we want.

    How do we actually use the fonts we selected on our web page?

    Well, we're getting to that.

    See that button called "Get embed code"? Go ahead and click on it to see the code we need to add to our CSS.

  7. Now you're taken to another screen that gives you code to include in your webpage to use the fonts you've selected.

    See the Web tab on the right side of the screen? That's where we're going to be working in.

    There are two radio buttons under there: one for <link> and one for @import.

    Go ahead and select the @import one.

  8. After selecting @import, the code sample beneath it will change.

    Now we have access to a CSS code snippet to include in our CSS...sort of.

    Under the section called "Embed code in the <head> of your html", see those <style> tags in there? That's HTML code, and we don't want that. However, the HTML <style> element is unique in that whatever code is put inside of it is processed as CSS.

    It's the code in between the <style> tags that is the CSS we want.

    Select and copy the code in between the <style> tags, and paste that at the top of our CSS file, AND MAKE SURE THERE'S A SEMI-COLON AT THE END!!!

    You NEED that semi-colon at the end; otherwise, it won't work at all, and will break other parts of your CSS!

    That @import code snippet will download the font we selected to our web page whenever anyone views it!

  9. However, we're not done yet.

    We've made the font available for us to use, but we haven't actually used it anywhere on the page yet.

    To do so, look at the code field(s) beneath the "Embed code in the <head> of your html" section; in there, you'll find the font-family property value that you'll want to use to apply that font to whatever element you want to set it to.

    You'll also find the appropriate font-weight and font-style properties specific to the font you selected (if applicable)

    You can use the CSS rule they give you as is, if you want to use that class in your HTML, but usually, we just care about the CSS properties for the particular font we want.

    For instance, if you wanted to apply the "Ms Madi" font to all paragraphs on your page, you could write something like this:

    p {
        font-family: 'Ms Madi', cursive;
        font-weight: 400;
        font-style: normal;
    }

    And that's basically it! You just need to include the @import at the top of your CSS file, and then set the font-family wherever you want to use it.

  10. But wait! There's more!

    What if you want to use more than one font from Google Fonts?

    Well, you can do that too!

    Let's check out another font from Google Fonts called Tektur.

  11. If you scroll down, you'll notice that Tektur has a LOT more Styles available for us to use.

    Click the "Get font" button again to add the font to our "shopping bag", and go inside it again.

  12. Once we go back to our "shopping bag", we see that the new font (Tektur) we selected is now in there too, along with Ms Madi.

    Click the "Get embed code" button again to get the updated code to use BOTH fonts on our web page.

  13. Back in this view, things have changed a bit: there are quite a few customization options for the Tektur font, which you can see on the left side.

    Modifying these will also modify the code on the right; however, it's probably best to just leave things at their default values. These settings will export all the font options for you, which is probably what you want.

    Once again, we're going to select the @import option, and pick the code from in between the <style> tags.

    Note that the code now includes references to both Ms Madi AND the Tektur font styles we selected.

    So if you want to use multiple Google Font font families on a web page, you don't need to have multiple @import lines of code; as long as you have them all grouped in the same "shopping bag", you should only need one single @import line to make them all available.

  14. Then, in the code beneath that, notice that we now have two separate font-family code snippets: one for Ms Madi and one for Tektur.

    Since Tektur has a lot more customization options, the CSS properties they include in its code snippet is also more complex. However, the most important part is still just the font-family; you can usually ignore the others.

    If you want to use Ms Madi in an element, then you'd pick the font-family: 'Ms Madi', cursive; code snippet.

    And if you want to use Tektur in an element, then you'd pick the font-family: 'Tektur', cursive; code snippet.

    And that's basically it! Just remember that none of the font-family code above or to the right will work unless the @import is at the top of your code.

Now you know how to include Google Fonts in your code! Have fun!
